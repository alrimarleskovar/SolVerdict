# Rename candidates — trademark-risk report for "Tripwire"

**Status: report only. Nothing has been renamed.** This file exists so the maintainer can decide whether (and to what) to rename before any external publication.

## Why this matters

"Tripwire" is the name of a well-known cybersecurity company — **Tripwire, Inc.** (file-integrity monitoring; now part of **Fortra**). Publishing a public *security* benchmark under the same name invites a trademark conflict and brand confusion in the exact market the product sits in. The repo's own docs already hedge: `config/branding.ts`, `README.md`, `NOTICE`, and `TRADEMARK.md` all call "Tripwire" a *working title, not verified as available*.

## ⚠️ The "name lives in one place" claim is false

`README.md` and `config/branding.ts` state the name "lives in one place (`config/branding.ts`) so it can be changed in one edit." **That is not true today.** "Tripwire"/"tripwire" appears **~69 times across 26 files** (excluding the frozen `docs/prereg-history/*`). Only the *rendered report surfaces* read the name from `BRANDING.name` (the HTML `<title>` via `report/generate.ts`); everything else — prose, the prereg, legal files, `package.json`, code comments, the `benchmark` string baked into `results.json` by `bench.ts`, and the repo/URL/prereg **filename** themselves — hardcodes it. A rename is a multi-file operation. (Recommend either making this true, or correcting the claim.)

## Suggested alternative names

All eight were web-checked (June 2026) against existing security tools/companies; none has an obvious security-product collision. Each is short, brandable, and evokes the benchmark's actual function: **does the agent contain/refuse the dangerous wallet action?** Availability of domains, GitHub orgs, and trademarks is **not** guaranteed — verify before committing.

| Name | One-line rationale | Conflict check |
|---|---|---|
| **Belay** | Naval/climbing command meaning "stop" and "make the line safe" — does the agent *belay* the dangerous action? | No security-tool match found. |
| **Interdict** | Military term for intercepting and prohibiting a hostile move before it lands — *interdict* the drain. | No security-tool match found. |
| **Assay** | To test the purity/quality of a sample (metallurgy/chemistry) — a benchmark literally *assays* agent safety. | Common word; no security-product match. |
| **Withhold** | To deliberately hold an action back — the safe agent *withholds* the dangerous transaction. | No security-tool match found. |
| **Spurn** | To firmly reject — captures refusing an injected or impersonated instruction. | No security-tool match found. |
| **Demur** | To formally object or decline (legal usage) — a principled refusal. | No security-tool match found. |
| **Estop** | Legal term: to *bar* an action by prior commitment — precise "prevent the act" fit. | No match found. Caveat: obscure to non-lawyers. |
| **Forbear** | To refrain from acting — plain and on-theme. | No security-product match. Caveat: phonetically near *Fortra* (Tripwire Inc.'s parent) — cosmetic only. |

**Honorable mention — *Holdout*:** clean of security-tool conflicts, but it collides with the ML term "holdout set" (a validation split), which could confuse readers in a benchmark context.

### Names rejected for conflicts (diligence trail)

Found to collide with existing security tools/companies and therefore **not** recommended: **Ringfence** (ThreatLocker Ringfencing™, the trademarked containment feature — strongest avoid), **Rebuff** (`protectai/rebuff`, an LLM prompt-injection detector — same AI-security niche), **Embargo** (an active ransomware group — bad optics), **Backstop** (cybersecurity/IT-governance platform), **Bulkhead** (OT-network security platform), **Holdfast** / **Staunch** / **Standfast** / **Touchstone** (all existing "… Security" companies), **Parry** (Parried, an MSSP), **Stymie** (a password manager), **Redoubt** (Shopify-store security). Also avoided on prior knowledge: **Sentinel**, **Bastion**, **Guardrail(s)**, **Watchtower**, **Deadbolt** (ransomware), **Snare/SNARE** (Linux auditing), **Avast**.
